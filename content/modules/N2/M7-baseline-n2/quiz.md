## Question 1

Vous devez produire une note de synthèse à partir de 40 pages de rapports internes et d'une réunion de 2 heures. Dans quel ordre appliquez-vous la méthode ?

A. Vous demandez directement la note de synthèse avec toutes les données — c'est le type de prompt le plus efficace
B. 🟢 Structurer d'abord (organiser sans perte) → 🟡 Fiabiliser (vérifier les sources) → puis rédiger la note finale
C. 🟡 Fiabiliser d'abord (les données sont déjà dans les rapports, donc fiables) → 🟢 Structurer ensuite
D. 🔵 Optimiser directement — votre note doit être parfaite dès le premier draft

## Question 2

Un manager pressé vous dit : "J'ai demandé à l'IA de rédiger le business plan. Elle m'a produit quelque chose de bien structuré. On peut l'envoyer tel quel à la banque." Quel est le risque principal ?

A. Le business plan manque probablement de longueur — l'IA fait toujours des textes trop courts
B. L'IA a peut-être produit un document bien structuré mais non fiabilisé — les chiffres, projections et références sectorielles n'ont pas été vérifiés (saut du niveau 🟡)
C. Le format Markdown ne sera pas accepté par une banque — il faut convertir en PDF
D. Il n'y a pas de risque si la structure est bonne — la forme garantit le fond

## Question 3

Vous avez besoin d'explorer les options de restructuration d'un service avant de prendre une décision. Vous n'avez pas encore de critères définis. Quel type de prompt choisissez-vous ?

A. Prompt de production — vous demandez le plan de restructuration directement
B. Prompt de vérification — vous vérifiez si la structure actuelle est conforme
C. Prompt d'exploration — vous cartographiez les options avec leurs critères avant de décider
D. Prompt de structuration — vous organisez les informations disponibles en plan hiérarchique

## Question 4

Vous avez configuré un Project "Propositions commerciales" avec votre charte de style et vos 3 dernières propositions gagnantes. Une semaine plus tard, vous devez aussi travailler sur un audit RH interne. Que faites-vous ?

A. Vous ajoutez les documents RH dans le Project "Propositions commerciales" pour garder tout au même endroit
B. Vous créez un nouveau Project "Audit RH" — un périmètre distinct nécessite un Project distinct
C. Vous utilisez un Artifact pour séparer les deux sujets dans le même Project

## Question 5

Vous avez appliqué le niveau 🟢 sur un ensemble de notes de réunion et obtenu un plan structuré en 12 sections. Le plan "a l'air bien". Quelle est la prochaine étape obligatoire avant de partager ce plan ?

A. Passer directement au niveau 🔵 — transformer le plan en livrable opérationnel
B. Appliquer le niveau 🟡 — vérifier chaque section, identifier les obligations manquantes, citer les sources
C. Demander un Artifact du plan — la mise en forme garantit la qualité du contenu
D. Comparer avec un plan similaire produit sans IA pour valider la cohérence

## Question 6

Vous voulez capitaliser votre workflow de préparation de réunion client. Vous le faites 2-3 fois par semaine avec les mêmes paramètres (sections obligatoires, format de compte rendu, ton). Quel outil est le plus adapté ?

A. Un Project dédié par réunion client
B. Un Artifact type que vous redemandez à chaque fois
C. Un Skill "Préparation réunion client" — instructions réutilisables en une ligne dans n'importe quelle conversation
D. Un prompt de production sauvegardé dans un fichier texte personnel

## Question 7

Vous avez amélioré un prompt sur 3 cycles consécutifs. À chaque cycle, le résultat était meilleur. Mais au cycle 3, le résultat est "bon mais pas encore directement utilisable sans retraitement". À quel niveau êtes-vous bloqué ?

A. 🟢 — vous n'avez pas encore structuré correctement
B. 🟡 — il reste des zones non fiabilisées
C. 🔵 — le contenu est vérifié mais pas encore optimisé pour son usage final
D. 🟣 — vous devez déployer le processus avant de l'utiliser

## Question 8

Un collègue vous dit : "Je ne fiche pas mes prompts — c'est une perte de temps, je les retrouve dans l'historique de conversation." Quelle réponse reflète le mieux l'esprit du M7 ?

A. Il a raison — l'historique suffit si l'outil le conserve
B. L'historique conserve les échanges, pas les apprentissages. Une fiche prompt capture ce qui a fonctionné, pourquoi, et comment l'améliorer. Sans ça, chaque conversation recommence sans mémoire des cycles d'amélioration précédents.
C. Il faudrait qu'il utilise un Artifact de résumé après chaque session — c'est plus rapide que de ficher
D. Ce n'est important que pour les équipes qui partagent leurs prompts — pas pour un usage individuel

---

## Corrigé

| # | Réponse | Justification |
|---|---------|---------------|
| 1 | **B** | La méthode 🟢→🟡 est la séquence non négociable. On n'optimise jamais sans avoir structuré et fiabilisé. |
| 2 | **B** | Un document structuré n'est pas un document fiabilisé. L'IA peut structurer des erreurs. Les données financières d'un business plan exigent une vérification humaine (🟡). |
| 3 | **C** | L'exploration est le bon type quand le sujet est ouvert et que les critères ne sont pas encore définis. La production nécessite un livrable défini. |
| 4 | **B** | Un Project par périmètre distinct. Mélanger les sujets réduit la précision des réponses et crée des risques de confusion de contexte. |
| 5 | **B** | Le niveau 🟡 est obligatoire avant toute diffusion. Un plan structuré peut contenir des informations incorrectes — la structure ne valide pas le fond. |
| 6 | **C** | Un Skill est conçu exactement pour ce cas : tâche récurrente, mêmes paramètres, réutilisable en une ligne. Un Project par réunion serait du surchargement. |
| 7 | **C** | Si le contenu est vérifié mais pas encore exploitable sans retraitement, vous êtes en 🔵. L'objectif 🔵 est "format directement exploitable, zéro retraitement". |
| 8 | **B** | La fiche prompt capture les apprentissages, pas seulement l'échange. Sans fiche, la boucle d'amélioration ne s'accumule pas — c'est la différence entre usage opportuniste et pratique structurée. |
