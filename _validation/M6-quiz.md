## Question 8

Votre responsable vous demande pourquoi vous avez du mal à obtenir des réponses précises de l'IA sur vos procédures internes. La cause la plus probable est :

A. L'IA n'est pas capable de traiter des documents internes d'entreprise
B. Vous utilisez un modèle généraliste qui ne connaît pas votre métier — vous n'avez pas fourni vos documents en contexte
C. Vous posez des questions trop longues qui dépassent la capacité du modèle
D. Les modèles IA ne peuvent traiter que des données structurées (tableaux), pas des documents texte

---

## Corrigé

| # | Réponse | Justification |
|---|---------|---------------|
| 1 | **B** | Les Projects conservent le contexte entre les sessions. Un nouveau prompt ou un Artifact ne suffit pas — ils ne persistent pas. |
| 2 | **B** | Un Project par périmètre de travail stable, avec les documents et instructions pertinents. Un Project par communiqué serait du surchargement inutile. |
| 3 | **B** | Un Artifact est un livrable structuré autonome — pas une réponse en prose, pas un prompt sauvegardé. |
| 4 | **C** | La formulation "Produis un tableau... format Markdown" demande un livrable directement exploitable. Les options A, B, D demandent des explications, pas un livrable. |
| 5 | **B** | Le RAG garantit l'accessibilité contextuelle, pas l'exactitude. La discipline 🟡 reste indispensable même avec une base documentaire connectée. |
| 6 | **B** | Un Skill est un ensemble d'instructions réutilisables, accessible depuis n'importe quelle conversation. Ce n'est pas une fonctionnalité technique avancée. |
| 7 | **B** | Un Project par périmètre distinct. Mélanger contrats fournisseurs et politique RH dans le même Project réduit la précision des réponses. |
| 8 | **B** | L'IA généraliste ne connaît pas vos documents internes. Il faut les fournir — dans un Project, via RAG, ou directement dans le prompt. |
