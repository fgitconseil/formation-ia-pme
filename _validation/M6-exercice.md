# Exercices — M6 Capitaliser sa connaissance
> 3 exercices pratiques · Durée totale estimée : ~45 min

## Exercice 1 — Créer votre premier Project métier (20 min)

### Objectif
Configurer un espace de travail persistant pour un périmètre réel de votre activité.

### Contexte
Vous avez vu comment les Projects permettent de ne plus re-expliquer le contexte à chaque session.
Cet exercice vous fait créer votre propre Project pour un périmètre que vous utilisez régulièrement.

### Instructions

**Étape 1 — Choisir votre périmètre (5 min)**

Identifiez un périmètre de travail où vous utilisez déjà (ou souhaitez utiliser) l'IA régulièrement.

Exemples selon votre rôle :
- Direction : "Comités de direction + reporting"
- RH : "Recrutement et offres d'emploi"
- Commercial : "Propositions commerciales + suivi client"
- Opérations : "Comptes rendus de réunion + plans d'action"

Écrivez votre périmètre choisi : _______________

**Étape 2 — Définir le contenu du Project (10 min)**

Pour votre périmètre, listez les 3 éléments à intégrer dans le Project :

1. **Instructions permanentes** (2-3 règles de travail) :
   - Exemple : "Toujours conclure par une liste d'actions avec responsable et date"
   - Votre règle : _______________

2. **Documents de référence** (1-2 documents que vous fournirez) :
   - Exemple : charte de marque, procédure interne, modèle de rapport
   - Vos documents : _______________

3. **Contraintes de format** (ce que l'IA doit toujours respecter) :
   - Exemple : "Format Markdown · Phrases courtes · Pas de jargon"
   - Vos contraintes : _______________

**Étape 3 — Tester le Project (5 min)**

Créez effectivement ce Project dans votre outil IA.
Envoyez une première demande réelle dans ce périmètre.

Notez la différence avec une conversation sans Project :
- La réponse était-elle plus précise ? _______________
- Avez-vous eu besoin de re-expliquer le contexte ? _______________

### Livrable
Un Project créé et une première conversation dans ce Project.

## Exercice 2 — Transformer une réponse en Artifact exploitable (15 min)

### Objectif
Reformuler une demande pour obtenir un livrable structuré directement utilisable.

### Contexte
La règle de l'Artifact : si vous allez retravailler manuellement la réponse avant de l'utiliser, demandez directement le livrable final.

### Instructions

**Étape 1 — Choisir votre document de travail (5 min)**

Identifiez un vrai document de travail que vous devrez produire prochainement.

Exemples :
- Compte rendu de la prochaine réunion d'équipe
- Brief pour un prestataire ou un recrutement
- Tableau de suivi d'un projet
- Checklist d'un processus récurrent (onboarding, bilan mensuel...)

Votre document choisi : _______________

**Étape 2 — Rédiger la demande d'Artifact (5 min)**

Rédigez votre prompt en demandant un Artifact structuré.

Utilisez ce gabarit :

```
Produis [type de document] en [format : Markdown / tableau / checklist].
Sections attendues : [listez vos sections]
Contraintes : [format, longueur, ton]
Le résultat doit être directement utilisable sans retraitement.
```

Votre prompt : _______________

**Étape 3 — Évaluer le résultat (5 min)**

Envoyez votre demande et évaluez l'Artifact reçu :

- [ ] Le format correspond à ce que vous avez demandé
- [ ] Il n'y a pas besoin de retraitement manuel avant utilisation
- [ ] Les sections sont dans l'ordre logique pour l'usage prévu
- [ ] La longueur est adaptée

Si un critère échoue : reformulez la contrainte correspondante et relancez.

### Livrable
Un prompt d'Artifact rédigé + le livrable obtenu évalué selon les 4 critères.
